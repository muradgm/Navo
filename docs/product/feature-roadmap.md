# Feature Roadmap

## Phase 0 — Prototype Extraction

- Convert Basel planner into generic data model.
- Replace hardcoded city logic with destination config.
- Keep mobile-first UI.
- Keep localStorage persistence.

## Phase 1 — MVP Trip Companion

- Trip setup wizard.
- Activity engine.
- Day builder.
- Weather-aware today plan.
- Budget planner.
- Food/essentials planner.
- Safety tab.

## Phase 2 — Personalization Engine

- traveler profiles
- trip purpose profiles
- preference scoring
- fatigue/energy model
- rainy/hot/cold day alternatives
- business time-block support
- kid-age suitability scoring

## Phase 3 — Data + Coverage

- city packs
- source registry
- opening hours source fields
- transport source fields
- official website and maps links
- generated/local image policy
- admin data editor

## Phase 4 — Accounts + Sync

- user accounts
- saved trips
- share trip plan
- cross-device sync
- offline mode/PWA

## Phase 5 — Monetization

Potential models:

- freemium trip limit
- premium smart planning
- affiliate links for tours/tickets
- business traveler plan
- family travel packs
- destination pack marketplace

Do not monetize too early. First prove repeat usage.
