# MVP Scope

## MVP Goal

Build a mobile-first trip companion that can handle one destination and one trip at a time with practical personalization.

## MVP Screens

1. Onboarding / Trip Setup
2. Today Plan
3. Activities
4. Day Builder
5. Food & Essentials
6. Weather
7. Budget
8. Safety / Practical Info
9. Saved / Favorites

## MVP Features

### Trip Setup

- destination city
- trip date range
- arrival/departure times
- base address
- traveler type
- children ages
- budget range
- food preferences
- transport mode
- activity interests

### Activity Discovery

- activity cards
- type filters
- cost filters
- kid-friendly filters
- weather filters
- energy filters
- duration filters
- distance/time notes
- external links
- maps links
- image gallery

### Day Builder

- selected day
- add/remove activity
- prevent duplicates
- estimated day cost
- recommended pacing
- warnings for overload

### Today Plan

- dynamic weather context
- family/business energy mode
- budget mode
- top recommendations
- avoid/reconsider section

### Food & Essentials

- breakfast/lunch/dinner strategy
- halal/vegetarian filters
- grocery map links
- nearby essentials
- lunch-only / cook-at-home modes

### Practical Safety

- emergency numbers
- hotel/base card
- route back to base
- day bag checklist
- lost child/business contingency notes

## MVP Data Model

Static data first:

- destinations
- activities
- grocery/essentials
- side trips
- trip preferences

Dynamic data:

- weather API
- localStorage saved plan

Backend can be added after the first validated prototype.
