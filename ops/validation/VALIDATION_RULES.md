# Validation Rules

## Global

- Product truth must be preserved.
- Core wedge must remain decision-focused, not list-focused.
- Mobile-first behavior must be preserved.
- No fake dynamic data.
- No hardcoded city-specific logic in generic components unless intentionally isolated in destination config.

## UI

- Buttons must be clickable and provide visible feedback.
- Cards must be expandable if they imply details.
- Empty states must be useful.
- Filters must update result count.

## Data

- Every activity must have title, summary, cost, duration, links, and at least two images for current visual requirement.
- Generated images must not be labeled as real photos.
- Google Maps links should exist for physical places.

## Budget

- Budget estimates must use EUR.
- If local currency is included, EUR still must be shown.
