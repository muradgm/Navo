# QA Review Checklist

## Product

- Does the change support the core wedge?
- Does it improve trip decision quality?
- Does it preserve mobile-first UX?
- Does it respect user context?

## Functionality

- Are buttons clickable?
- Is feedback visible?
- Does localStorage persist where expected?
- Does filtering work?
- Does budget update correctly?
- Does weather fallback behave honestly?

## Data

- Are activities mapped to correct images?
- Are links present and useful?
- Are cost estimates in EUR?
- Are source limitations clear?

## Regression

- Existing itinerary builder still works.
- Favorites still work.
- Language toggle still works.
- Weather does not crash offline.
